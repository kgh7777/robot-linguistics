Sensor Grounding Pipeline

Live demonstration of sensor-grounded essay synthesis — a first step toward Robot Linguistics.

Core pipeline:

[Physical sensor event] (Radeon 48.0°C, Battery 10.5V, Motor 85°C)
       ↓
[External function] (compose_essay_universal) — deterministic, no LLM call
       ↓
[English essay] (3-paragraph structure: Introduction + Body + Conclusion)

Result: The Qwen2.5 LLM weights (trained by Chinese programmers) are bypassed. The essay is generated by a deterministic function that uses only the live sensor measurements and the Qwen-trained English vocabulary. This is the causal grounding of language in physical events.

1. What is Sensor Grounding?

When an LLM says "The temperature is 48.0°C, Move quickly," two questions arise:

  • Where did the number 48.0°C come from? → From rocm-smi reading the physical sensor.
  • Where did the words "temperature", "Move quickly" come from? → From Qwen's training data (via Chinese programmers).

This pipeline separates the two: the measurement is real-time and physical, while the vocabulary is borrowed from the LLM's training. The combination yields causally grounded language.

4. From Sensor Grounding to Robot Linguistics

This pipeline demonstrates a single principle: a robot's utterance does not require philosophical meaning. Causal grounding is sufficient.

Drawing on Halliday (1978, 1985) and the symbol grounding problem (Harnad 1990; Bender & Koller 2020), we propose a working outline of Robot Linguistics:

4.1 Two registers of robot language

  • Robot Language (Robot-to-Robot): Direct sensor-data representation, e.g. GPU.Temperature=48C; Battery.Voltage=10.5V; Motor.RPM=3000
  • Robot-Human Common Language: Natural English/Korean utterances like "The temperature is 48.0°C, Move quickly."

4.2 Four parts-of-speech grounded in physics

  • Noun = physical event (temperature, voltage, distance, time)
  • Verb = physical action (Move, Stop, Reduce, Increase)
  • Adjective = physical state (High, Low, Stable, Critical)
  • Adverb = physical quantity (Quickly, Slowly, Steadily)

4.3 Speech acts as physical-act language

Speech act Example Grounded in
Declarative "The temperature is 48.0°C" Sensor reading
Imperative "Move quickly" Action mapping
Conditional "If the temperature is high, Move quickly" Threshold + Action
Concessive "Although the temperature is high, the system is stable" Stability assessment

4.4 Multi-agent robot-to-robot communication

A Qwen instance measuring a GPU could emit "Temperature rising rapidly, Move quickly." A second LLM (controlling a motor) could parse this and issue a cmd_vel command. This is causally grounded inter-robot language.

6. Reproducing the live demonstrations

6.1 Single-domain, single-timestamp (4 cases × 3 domains = 12 combinations)

python3 -c "
import sys; sys.path.insert(0, '/home/kgh')
import gpu_telemetry as gt
import importlib
gp = importlib.import_module('grounding_pipeline01')

# Battery essay — value 12.4V, comparison '<=', threshold 11.0
readings = [gt.GpuSample(0, 'battery1', 'mock', temperature_c=12.4, power_w=0.0)]
thresholds = [gt.Threshold('temperature', '<=', 11.0, 'Return to base')]
print(gp.compose_essay_universal(readings, thresholds))
"

6.2 Multi-domain timeseries essay (3D)

python3 -c "
import sys; sys.path.insert(0, '/home/kgh')
import gpu_telemetry as gt
import importlib
gp = importlib.import_module('grounding_pipeline01')

gpu_history = [[gt.GpuSample(0, 'card0', 'rocm-smi', temperature_c=44.0, power_w=20.0)],
               [gt.GpuSample(0, 'card0', 'rocm-smi', temperature_c=46.0, power_w=22.0)],
               [gt.GpuSample(0, 'card0', 'rocm-smi', temperature_c=48.0, power_w=25.0)]]
gpu_thresholds = [gt.Threshold('temperature', '>=', 40.0, 'Move quickly')]

battery_history = [[gt.GpuSample(0, 'battery1', 'mock', temperature_c=12.5, power_w=0.0)],
                   [gt.GpuSample(0, 'battery1', 'mock', temperature_c=12.3, power_w=0.0)],
                   [gt.GpuSample(0, 'battery1', 'mock', temperature_c=12.4, power_w=0.0)]]
battery_thresholds = [gt.Threshold('temperature', '<=', 11.0, 'Return to base')]

motor_history = [[gt.GpuSample(0, 'motor2', 'mock', temperature_c=30.0, power_w=0.0)],
                 [gt.GpuSample(0, 'motor2', 'mock', temperature_c=32.0, power_w=0.0)],
                 [gt.GpuSample(0, 'motor2', 'mock', temperature_c=35.0, power_w=0.0)]]
motor_thresholds = [gt.Threshold('temperature', '>=', 80.0, 'Stop motor')]

history = {'gpu': gpu_history, 'battery': battery_history, 'motor': motor_history}
thresholds = {'gpu': gpu_thresholds, 'battery': battery_thresholds, 'motor': motor_thresholds}

print(gp.compose_essay_timeseries_multidomain(history, thresholds))
"
